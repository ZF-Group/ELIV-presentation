## Structure of this repository
- /build: Storage of build scripts, makefile, build relevant tool-configurations (e.g. Linker-Definitions)
- /design: Storage of files related to the software design (e.g. UML diagrams)
- /doc: Storage of SW-documentation
- /include: Storage of public header files for e.g. 
- /output: Storage of build-results, hex-files etc. (content not kept)
- /res: Storage of static resources e.g., images
- /src: Storage of source files, sub-directories can be created for structuring e.g., /lib, /app, /src
- /test: Storage for module tests, sw-tests scripts etc. (separated from \source)
- /tools: Storage for additional tools and (not build relevant) tool-configurations (no binary files allowed)
